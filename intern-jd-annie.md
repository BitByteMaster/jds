## 📈 项目二：固收产品的 RWA（真实世界资产代币化）

> **核心标签**：固定收益、RWA、收益分配模型、产品设计
> **安妮的优势适配**：这直接对应了 Finance 和 Data Science 的结合。她需要将复杂的金融模型抽象为业务逻辑和代码逻辑。

您提供的三种收益分享模式（Profit Sharing Models）非常有代表性，我们可以帮她进一步**标准化和金融工程化**：

### 1. 三种模式的完善建议

* **模式 1：On Fix (High Interest Like 10% Or Above)**
* *完善方向*：定义为**固定收益凭证（Fixed Yield Vault）**。需要安妮测算底层 RWA 资产（如美债、供应链金融）如何支撑 ≥10% 的高息，并设计底层利差（Spread）的风险储备金机制。


* **模式 2：By Introducing Equal Capital and Sharing Equally (Partnership Model)**
* *完善方向*：定义为**合伙人风险共担模型（Equity-like/LP-GP Model）**。同股同权，利润与亏损按比例均摊。需要安妮设计清算（Liquidation）水位线和净值（NAV）更新逻辑。


* **模式 3：APY Cost Plus 30% In Profits Only (No Loss Sharing)**
* *完善方向*：定义为**保本非固定收益模型（Principal Protected with Upside Capture）**。用户拿走基础 APY，超额利润中平台扣除 30% 表现费（Performance Fee）。需要安妮设计高水位线（High-Water Mark）计算公式，确保平台只在真正赚钱时抽成。



### 2. 预期产出

* 《RWA 固收产品收益模型数学公式与清结算逻辑文档（PRD）》。
* （若她会 Python/SQL）协助后端或智能合约团队，用代码实现上述三种模式的利息与利润分配算法。

## 🤖 项目三：量化交易 AI 多智能体（Multi-Agent）系统搭建

> **核心标签**：大语言模型（LLM）、AI Agent、量化交易、金融数据科学
> **安妮的优势适配**：这是 EFDS 中 "Data Science" 的天花板级别应用。IC 的学生通常对 Python 极其熟练，且具备基本的统计和机器学习知识。

由于实习时间有限，建议让她基于现有的开源框架（如 **LangGraph, AutoGen 或 CrewAI**）来搭建一个极简的量化研究 Agent 团队。

### 1. 多智能体分工架构设计

可以让安妮尝试搭建一个包含以下角色的系统：

1. **宏观/新闻分析 Agent (Researcher Agent)**：负责抓取和分析市场新闻、推特情绪、美联储决议（利用 LLM 进行情绪打分 $Sentiment \ Score \in [-1, 1]$）。
2. **技术面分析 Agent (Technical Analyst Agent)**：利用 Python 计算常规指标（MACD, RSI, 波动率），识别趋势。
3. **风控与投资组合 Agent (Risk/Portfolio Agent)**：结合前两个 Agent 的输入，根据凯利公式（Kelly Criterion）或马科维茨模型计算仓位配比与止损点。

### 2. 预期产出

* 在本地或本地服务器跑通一个“多智能体复盘/模拟交易”的 Python 脚本。
* 《AI Multi-Agent 系统在量化策略中的回测表现与可行性评估报告》。
